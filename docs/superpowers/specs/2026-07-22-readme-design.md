# README Design

## Goal

Replace the minimal README with a practical setup guide for the local RAG workspace.

## Content

The README will cover the application purpose, principal capabilities, required local tooling, `.env` configuration, model preparation, Docker Compose startup, local URLs, development test commands, and a concise service map. It will link to `.env.example` rather than duplicate every variable and will avoid operational detail not required to run the project.

## Verification

The documented commands and URLs will be checked against the repository configuration before the README is committed.
